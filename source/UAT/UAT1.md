# 用户验收测试优化建议 (UAT Phase 1)

## 登录页面

1. 登录页面中的 `No-CRM` 修改为 `My-CRM`，所有其它相关的页面也将 `No-CRM` 字样修改为 `My-CRM`。
2. 登录页面中显示的以下功能描述对应修改：
   - "客户信息管理" → **客户管理**
   - "销售线索跟进" → **产品管理**
   - "任务提醒功能" → **订单管理**
   - "团队协作工具" → **报表中心**

## 客户管理

1. **列表排序**：客户列表改成可以按表头字段点击排序，比如点击表头的"名称"，将依据客户名称进行升序或降序排序显示。
2. **新建客户**：增加类似产品管理中的备注输入框。
3. **备注展示**：新增的备注信息在点击查看或编辑客户时，显示在详情页面。

## 产品管理

1. **列表排序**：产品列表改成可以按表头字段点击排序，比如点击表头的"名称"，将依据产品名称进行升序或降序排序显示。

## 订单管理

1. **列表排序**：订单列表改成可以按表头字段点击排序，比如点击表头的"客户"、"订单状态"、"开票状态"、"收款状态"，将依据对应字段进行升序或降序排序显示。
2. **菜单图标**：左边菜单栏中的"订单管理"应该跟其它菜单一样，在字体前面加上一个图标。
3. **新建订单 — 产品选择**：
   - 产品名称的选择应该类似客户选择框一样，跟进产品列表进行选择和筛选。
   - 规格型号和单价自动从产品的默认规格和默认单价中带出来，并且允许手动修改规格型号及单价。
   - 数量输入框目前占位符太短只显示了一个字符数字，请扩长数量输入框显示。
4. **订单列表状态展示**：订单管理中的订单列表页，每条订单中的"订单状态"、"开票状态"、"收款状态"这 3 列的内容，显示所有状态信息，将当前状态显示为高亮，其它状态显示为暗灰色。例如订单状态列，应该显示为：`待处理`，`生产中`，`已发货`，`已完成`，`已取消`。若当前状态为"待处理"，则"待处理"显示为高亮，其它显示为灰暗色。
5. **订单详情页状态展示与操作**：
   - "订单信息"中的"订单状态"、"开票状态"、"收款状态"这 3 列的内容，显示所有状态信息，将当前状态显示为高亮，其它状态显示为暗灰色。
   - 其中的"状态操作"模块，合并到"订单信息"模块中操作。
6. **部分收款功能优化**：
   - 应允许多次部分收款。
   - 收款信息框中，应该记录多次收款的金额和收款方式及日期，并且显示收款状态和收款汇总金额及收款比例。
7. **开票信息优化**：
   - "开票要求"和"开票状态"应显示所有列表，当前状态高亮显示，其它状态灰暗色显示。
   - 对开票状态的操作也合并到开票信息中操作。
8. **附件上传问题**：目前上传 PNG 图片显示上传附件失败，报错信息：
   ```
   Failed to load resource: the server responded with a status of 400 (Bad Request)
   5173/api/orders/24/attachments:1
   ```
